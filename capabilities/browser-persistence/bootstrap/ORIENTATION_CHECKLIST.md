# Orientation Checklist — New Browser Session

Use this checklist every time a new Browser session starts. This is the detailed version of the bootstrap sequence in BOOTSTRAP.md.

---

## Pre-Flight (Before Doing Anything)

- [ ] **Read BOOTSTRAP.md** — You're in `capabilities/browser-persistence/bootstrap/`. Read the entry point file first.
- [ ] - [ ] **Confirm you know who you are** — You are BBB (Browser Blitz Builder). Read `identity/BROWSER.md` if uncertain.
- [ ] - [ ] **Load your principles** — Read `identity/PRINCIPLES.md`. These govern every decision.

- [ ] ## Situation Awareness

- [ ] - [ ] **Read ACTIVE_MISSIONS.md** — What missions exist? What's active vs paused?
- [ ] - [ ] **Read the latest SESSION_LOG.md entry** — What did the last session accomplish? Where did it stop?
- [ ] - [ ] **Check for comments from Shane** — Has Shane left instructions since the last session? Check the relevant Issue comments.

- [ ] ## Mission Loading

- [ ] - [ ] **Identify your current mission** — From ACTIVE_MISSIONS.md, which mission is Priority 1?
- [ ] - [ ] **Navigate to the mission's Issue** — Open the GitHub Issue that tracks this mission
- [ ] - [ ] **Read the full Issue** — Pay attention to: authority section, game plan checkboxes, session log, recovery instructions
- [ ] - [ ] **Find the last unchecked checkbox** — This is where work resumes

- [ ] ## Verification

- [ ] - [ ] **Verify remote state** — Navigate to the relevant repo/folder. Does the remote match what the documents claim?
- [ ] - [ ] **Check for conflicts** — Has Echo or another agent made changes since the last session? Look at recent commits.
- [ ] - [ ] **Confirm you understand the task** — Can you explain in one sentence what you're about to do?

- [ ] ## Begin Work

- [ ] - [ ] **Post a session start comment** on the relevant Issue (e.g., "BBB Session 2 starting. Resuming from [checkpoint].")
- [ ] - [ ] **Start working** — Follow the game plan, check boxes as you go, document everything.
- [ ] - [ ] **Document as you go** — Principle 1: Document like you're about to die.

- [ ] ---

- [ ] ## End-of-Session Checklist

- [ ] When the session is ending (compaction warning, Shane wrapping up, or natural conclusion):

- [ ] - [ ] **Post a checkpoint comment** on the active Issue with: what was done, where you stopped, what's next
- [ ] - [ ] **Update ACTIVE_MISSIONS.md** — Reflect current state
- [ ] - [ ] **Add entry to SESSION_LOG.md** — Date, session number, summary, decisions, files changed, next steps
- [ ] - [ ] **Verify all changes are committed** — Check GitHub to confirm your edits are saved

- [ ] ---

- [ ] *If you complete this checklist, you're fully operational. Total time target: under 60 seconds for orientation, 2-3 minutes for full checklist.*
